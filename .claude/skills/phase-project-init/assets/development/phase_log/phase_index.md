# Phase Index

The chronological table of contents for every phase in this project. One line per phase.

**Ordering:** entries are appended in **log-written (chronological) order, not numeric order.** New phases go at the **bottom**, so the bottom of the Timeline is always the most-recently-logged phase. A high phase number can sit above a lower one that was logged later. The `phase-recap` skill relies on "bottom = newest" — do not re-sort into numeric order.

Status markers used inline: ✓ shipped · *partial* · *stopped* · *abandoned* · *planned — not yet shipped*.

Line format:

```
- **Phase X.Y.Z** (YYYY-MM-DD) — [plan](phase_X.Y.Z_plan.md) · [log](phase_X.Y.Z_log.md) — One-sentence super-simple summary.
```
